Arrays and Hashes

Comparing Ruby's Collection Objects

November 15, 2015

What's an array?

An array is an ordered collection of objects. Any object at all can be stored in an array: floating point numbers, integers, symbols, other arrays, hashes, file names, classes, and boolean values. Really, an array is an ordered collection of object pairs in which one of the objects is always an integer; that integer corresponds to its object's position in the array. Arrays are indexed from left to right, starting from 0, and elements are accessed by their index number. So if you want to access the first element of an array, you type array[0], for the second element, array[1], for the hundreth element, array[99], and so on. It's worth noting here that the contents of an array always remain in the same order until you explicitly move them by calling one of the array methods.

Here's an example of an array of strings assigned to the variable "city_names":

city_names = ['Beijing', 'Chicago', 'Nashville', 'Detroit']

To access 'Chicago' in the city_names array, type: city_names[1]

What's a hash?

Hashes are a lot like arrays, except objects are indexed differently in hashes. Instead of indexing objects with integers in order from left to right, hashes organize objects in key: value pairs. You access a value by referring to its key; a key can be a string, integer, or any other kind of object. Here's an example of a hash:

state_abbreviations = {

'IL' => 'Illinois',

'NY' => 'New York',

'WI' => 'Wisconsin',

'TX' => 'Texas'

}

If you want to return the value 'Wisconsin', you refer to its key, like this:

state_abbreviations['WI']

When to use arrays. When to use hashes.

As you can tell, arrays and hashes are quite similar, so a natural question is which one is better? The obnoxious answer is 'both' and 'neither'. You could argue that hashes are more work to construct by hand because you have to type in a key for each value, whereas arrays automatically index their elements. However, it might be nice to be able to access your values by typing a short string, as in the state abbreviations example. Your choice of collection object will ultimately depend on your coding needs and style.
