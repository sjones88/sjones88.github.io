The Enumberable#map Method

November 23, 2015

Enumberable#map: What is it?

If you've ever iterated over an array in Ruby, then you've probably encountered the each method. If you have an array of strings that you want to print to the screen, the each method is your friend. You would probably do something like this:

south_park_kids = ['Cartman', 'Kyle', 'Stan', 'Kenny']

south_park_kids.each {|kid| puts kid}

Printing lists to the screen is great, but what if we want to do something really exciting with our collection objects? Let's say we have an array of integers and we want to return an array of slightly bigger integers (bigger is always better, right?). Your first instinct might be to use each and point a variable at the result. You might try something like this:

small_integers = [1,2,3,4,5]

embiggened_integers = small_integers.each {|small_int| small_int * 2}

return embiggened_integers

You would be disappointed if you expected this code to return [2,4,6,8,10]. It turns out that each always returns its receiver object, so when you pointed embiggened_integers at small_integers.each... Ruby returned the same thing as embiggened_integers = small_integers. That's because Enumerable#each lacks a way to save values as it iterates through a collection. If only there were an enumerable method that worked a lot like each while being clever enough to save the results of its work...

Enter the map method

The map method is Ruby's answer to your plea. It iterates and calls its block on a collection, just like each, but it saves the results and returns a new array. Brilliant! Let's try embiggening those integers again:

small_integers = [1,2,3,4,5]

embiggened_integers = small_integers.map {|small_int| small_int * 2}

return embiggened_integers

Positively cromulent!
